# Gmail Suspicion Meter

Gmailのメール一覧に「怪しさ%」を表示するChrome拡張です。AIへのプロンプト生成、ChatGPT/Geminiへの遷移、AI結果の貼り付け反映、圧縮記憶による次回以降の%表示に対応しています。

## 使い方

1. Chromeで `chrome://extensions/` を開きます。
2. 右上の「デベロッパーモード」をオンにします。
3. 「パッケージ化されていない拡張機能を読み込む」を押します。
4. このフォルダ `gmail-suspicion-extension` を選択します。
5. Gmailを開き、拡張機能アイコンを押すとパネルが表示されます。
6. 「ChatGPTで判定」または「Geminiで判定」を押します。
7. プロンプトがクリップボードにコピーされ、AI画面へ遷移します。
8. AIのJSON結果をパネル内の生成結果欄に貼り付け、「反映」を押します。

## ChatGPT遷移仕様

- プロンプトが9000文字以内の場合:
  - `https://chatgpt.com/?temporary-chat=true&prompt=XXX`
- 9000文字を超える場合:
  - `https://chatgpt.com/?temporary-chat=true`
- どちらの場合もプロンプトはクリップボードにコピーされます。

## 解析済みメールだけに表示する仕様

この版では、AI判定に使ったメールだけを解析済みとして記憶します。

GmailのメールURL末尾が次のような形式の場合、

```text
https://mail.google.com/mail/u/0/?tab=rm&ogbl#inbox/FMfcgzQgLrwMshVKkVfdRDQdcmpPlkMR
```

`FMfcgzQgLrwMshVKkVfdRDQdcmpPlkMR` の先頭文字を短いキーとして保存します。
初期値では先頭3文字だけを保存します。

```js
const MAIL_KEY_PREFIX_LENGTH = 3;
```

未解析のメールには怪しさ%を表示しません。
解析済みキーに一致したメールだけ、保存済みルール・ドメイン・送信者情報を使って怪しさ%を表示します。

先頭3文字は情報量が少ない一方で、別メールと衝突する可能性があります。衝突を減らしたい場合は、`content.js` の `MAIL_KEY_PREFIX_LENGTH` を8〜10程度に変更してください。

## 記憶する情報

メール本文そのものは保存しません。保存対象は次のような圧縮された特徴のみです。

- 解析済みメールの短いキー
- ドメイン別スコア
- 送信者別スコア
- 短いキーワードルール
- 件名・送信者に対する短い特徴ルール

## 注意

GmailのDOM構造に依存するため、Gmail側のUI変更でメール行の取得や表示位置がずれる可能性があります。

## 2026-05-26 修正

前版では、メールURL末尾トークンが取得できない行をプロンプト対象から除外していたため、Gmail環境によっては `今回の表示メール一覧: []` になる場合がありました。

この版では、一覧取得部分を旧バージョン寄りに戻し、URLトークンが取れない行もAI判定対象に含めます。
URLトークンが取れる場合はその先頭3文字を解析済みキーに使い、取れない場合は一覧に表示されている送信者・件名・短いスニペットから作った短いフォールバックキーを使います。
